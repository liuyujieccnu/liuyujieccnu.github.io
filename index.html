  • 有“医美”(仅指注射类产品，而非手术类）需求的==人群普遍存在误区==

具体改进意见

1. 什么是调研？调研——数据与结果说话

重要：调研突出的是自己实际行动的调研不是编故事

  • 目前情况：无数据、无依据、空口白话（人尽皆知的大白话）、不专业。请注意比赛的性质是专业性的商业性的。
  • 建议：行业报告截图支撑、查阅论文理论依据支撑，用具体数据说话，但是不宜过多（艾尔建作为专业的机构，已经对数据很了解，多了一是没必要，二是容易错别人很快会做出专业的判断），还可以自行统计某段时间新闻报道次数，主要问题领域等。
  • 目前情况：远离生活，虚无缥缈。
  • 建议：生活中（尤其是校园）的调研问卷，访谈记录，街头采访视频。有访谈案例更好，不一定是视频可以是文字记录。

2. 什么是人群普遍误区？就是要找出目标人群及需解决的问题。

  • 目前情况：与调研脱钩、不精确、废话为主。解决问题不是要解决所有人的问题，能解决相当一部分人群的问题，方案就是成功的。很多人生活的领域以及求美心态是离我们比较远，我们难以去深入了解和采取行动的。

  • 建议：结合我们的自身情况。那么我们的自身情况是什么：

  1. 大学生，更准确的来说是师范大学的大学生，师范学校的特点你们更清楚我不过多赘述。
  2. 互联网络的使用者，微博你们用的太多了。
  3. 那么我们最能接触到的人，最能产生同理心的人是怎样一群求美者？首先是生活在师范学校女学生其次是活跃在互联网络上的同龄人
  • 结合我们的情况，那么她们的问题是什么？你们应该比我清楚（一定要结合调研数据）。请注意采用分散——总结的结构或者总结——分散的结构模式。

举例：调研发现喜欢自己网上买一些廉价产品自己使用等（如注射类）。
结合自己平时在使用美容产品中遇到的问题。

3. 什么是解决方案？创新解决方案突出的是创新二字。

  • 目前情况：
  1. 解决方案可以说是这个方案最重要的地方，总共用了三句话，需要明白的是：前面是在回答为什么要做这个解决方案，为解决方案铺路，而解决方案仅仅只有三句话。
  2. 可持续性不强，操作性有待提高。
  • 建议：
  1. 方案具体化，要从自己的视频中走出来，站在一个更高的高度，不是说我做了一个视频这么简单。如果就视频说视频何来方案可言。
  2. 形成解决方案体系，活动体系。
  3. 视频（就是目前PPT中说的新闻截取那些）知识活动的第一步活动的引子，整个方案是一个能解决问题的体系。

方案举例（前提：我们前面已明确目标人群，方案以校园活动和online&offline结合的活动为主）。

  1. 推广视频，展开话题。（这个一定是第一步我是以视频制作为基础的）
  2. 在校内：联合学校组织（如：计算机学院、教育信息技术学院）开展相关动画传媒作品制作比赛。联合校医院、艾尔建等机构在校内开展医疗美容专题讲座。还可以再思考一些其他的创新活动机制。
  3. 形成医美知识普及的常态化机制，推广到其他女生较多的学校。
  4. 在学校以外，针对互联网络使用的同龄求美者，联合本地媒体，开展微电影拍摄，地方台栏目剧发起活动，主角寻找等活动，当然，这只是我的基础想法，你们可以思考更多的解决方案。
    但是不能忽略的是方案的整体性、连贯性、可操作性。

CommonJS模块规范（module.exports与require）

每个文件就是一个模块，有自己的作用域。在一个文件里面定义的变量、函数、类，都是私有的，对其他文件不可见。

module.exports

CommonJS规范规定，每个模块内部，module变量代表当前模块。这个变量是一个对象，它的exports属性（即module.exports）是对外的接口。加载某个模块，其实是加载该模块的module.exports属性，后缀名默认为.js。

require

require方法用于加载模块。

require命令的基本功能是，读入并执行一个JavaScript文件，然后返回该模块的exports对象。如果没有发现指定模块，会报错。

目录加载规则

require发现参数字符串指向一个目录以后，会自动查看该目录的package.json文件，然后加载main字段指定的入口文件。如果package.json文件没有main字段，或者根本就没有package.json文件，则会加载该目录下的index.js文件或index.node文件。

Jasmine的安装

在目标文件夹下

配置Jasmine.json

可以运行jasmine examples获得一个实例，然后替换相关文件

  • spec_dir：根目录（下面两个参数的前缀），下面的spec_files和helpers都是相对于spec_dir的。
  • spec_files: 测试文件所在位置,这里的”*/[sS]spec.js”表示在目录spec下面的所有后缀名为spec或Spec的js文件。
  • helpers: 存放正式js文件，就是我们测试的目标文件，spec_files就是用来测试helpers的，这里同样是相对路径（相对spec_dir）
  • stopSpecOnExpectationFailure:这个参数从字面意思上是“当一个spec出现异常时是否停止该spec”，实际上也就是这个意思，一个spec中可以有多个expect,true为停止，false为不停止。

Jasmine测试spec

describe('your test name',function(){
    it("contains spec with an expectation", function() {
       expect(true).toBe(true);    //true==true为true，正确
    });

    it("the second it", function() {
       expect(true).toBe(false);  //true==false为false，会报错
    });
});

Jasmine被测试的文件

在被测试的文件的末尾加上module.exports = function_name;
就是在模块最后写好模块接口，声明这个模块对外暴露什么内容。
